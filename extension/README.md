# Локальный мост DeepSeek (расширение)

Расширение связывает браузерный DeepSeek и локальный Python-клиент по WebSocket.

## Что делает расширение

- Подключается к локальному WebSocket Python-клиента.
- Принимает задачи (`deepseek_task`) от Python.
- Открывает/использует вкладку DeepSeek, отправляет запрос и получает ответ.
- Возвращает результат в Python (`deepseek_task_result`).
- Обрабатывает по одной задаче за раз; остальные ставит в очередь (FIFO).

## Установка

1. Открой `chrome://extensions` (или `edge://extensions`).
2. Включи **Режим разработчика**.
3. Нажми **Загрузить распакованное расширение**.
4. Выбери папку `extension`.
5. Закрепи иконку расширения в панели браузера.

## Панель управления (веб-интерфейс)

Нажми на иконку расширения — откроется вкладка с панелью настроек (если панель уже открыта, фокус перейдёт на неё).

На панели сразу доступны:

- **Адрес** (хост Python-сервера, обычно `127.0.0.1`)
- **Порт** (обычно `8765`)
- кнопка **Соединить** / **Отключить**
- блок **Селекторы DeepSeek** (поле ввода, кнопка отправки, блок ответа, цитаты)
- кнопки **Проверить**, **Сохранить**, **Сбросить** для селекторов

### Поведение

- **Соединить**:
  - применяет адрес/порт;
  - включает мост и запускает подключение к Python;
  - после запуска адрес/порт становятся недоступны для изменения.
- **Отключить**:
  - отключает мост;
  - останавливает переподключение;
  - снова разрешает редактировать адрес/порт.

## Настройки селекторов DeepSeek

### Кнопки

- **Проверить** — проверяет селекторы на открытой вкладке `https://chat.deepseek.com/*`.
- **Сохранить** — сохраняет селекторы в `chrome.storage.local`.
- **Сбросить** — возвращает селекторы по умолчанию.

### Подсветка при проверке

При **Проверить** расширение:

- выделяет найденные элементы яркой красной рамкой;
- показывает подписи на странице:
  - `поле ввода`
  - `кнопка отправки`
  - `блок ответа`
  - `цитаты`

### Что вставлять в поля

Вставляй **CSS-селектор**, а не весь HTML.

Как получить:

1. Открой DevTools -> `Elements`.
2. Выбери нужный элемент.
3. `Copy -> Copy selector`.
4. Вставь селектор в нужное поле.

Если у тебя только `outerHTML`, выдели в нем стабильные class/атрибуты и собери CSS-селектор.

## Очередь

- Одновременно выполняется **одна** задача (`MAX_PARALLEL_TASKS = 1`).
- Пока она идёт, новые запросы от Python **ждут** в FIFO-очереди.
- После завершения текущей задачи следующая из очереди запускается автоматически.

## Логи и отладка

Открой DevTools сервис-воркера расширения в `chrome://extensions` -> расширение -> **service worker**.

Ключевые логи:

- `[DeepSeek][ws] попытка подключения ...`
- `[DeepSeek][ws] подключено`
- `[DeepSeek][bridge] задача получена`
- `[DeepSeek][bridge] задача поставлена в очередь`
- `[DeepSeek][bridge] задача запущена`
- `[DeepSeek][bridge] результат отправлен в Python`
- `[DeepSeek][bridge] задача завершена`

## Формат обмена с Python

### Вход в расширение

```json
{
  "type": "deepseek_task",
  "requestId": "req-...",
  "payload": {
    "task": "Текст задачи",
    "timeoutMs": 180000
  },
  "timeoutMs": 180000
}
```

Если в `payload` передать `"reuseDeepseekTab": true` (так делает Python при `reuse_deepseek_tab=True`):

- после ответа вкладка DeepSeek **не закрывается**;
- следующая задача продолжает **тот же чат**;
- при первом запросе в этом режиме вкладка создаётся и чат очищается как обычно; дальше сообщения идут подряд в одном диалоге.

При `reuseDeepseekTab` без флага или `false` после задачи вкладка закрывается; если ранее была открыта вкладка в режиме reuse — она закрывается перед новой одноразовой задачей. При отключении моста в панели расширения сохранённая вкладка reuse тоже закрывается.

### Закрытие reuse-вкладки из Python

Клиент с `reuse_deepseek_tab=True` может отправить по WebSocket:

```json
{
  "type": "deepseek_close_reuse_tab",
  "requestId": "req-..."
}
```

Ответ в Python:

```json
{
  "type": "deepseek_close_reuse_tab_result",
  "requestId": "req-...",
  "ok": true
}
```

При ошибке: `ok: false`, поля `error` и `message`.

### Успешный ответ в Python

```json
{
  "type": "deepseek_task_result",
  "requestId": "req-...",
  "ok": true,
  "result": "Итоговый текст",
  "parsedJson": {},
  "sources": []
}
```

### Ответ с ошибкой

```json
{
  "type": "deepseek_task_result",
  "requestId": "req-...",
  "ok": false,
  "error": "deepseek_error",
  "message": "Описание ошибки"
}
```

## Где хранятся настройки

`chrome.storage.local`:

- `bridgeEnabled`
- `bridgeWsHost`
- `bridgeWsPort`
- `bridgeDeepseekSelectors`
- `bridgeReuseDeepseekTabId` — id вкладки DeepSeek в режиме `reuseDeepseekTab` (чтобы после сна сервис-воркера MV3 снова найти ту же вкладку, пока она открыта). Сбрасывается при закрытии вкладки, отключении моста или сообщении `deepseek_close_reuse_tab` из Python.

## Ограничения

- Расширение работает с `https://chat.deepseek.com/*`.
- Для корректной работы пользователь должен быть авторизован в DeepSeek.
- Изменения в верстке DeepSeek могут требовать обновления селекторов.

### Медленная загрузка страницы

Перед отправкой промпта расширение ждёт, пока у документа будет `document.readyState === 'complete'` (опрос до ~15 с). Это **не** равно «готов интерфейс чата»: SPA может дорисовать поле ввода позже.

Если селектор из настроек ещё не совпал, код может взять **первое видимое** поле ввода на странице (`textarea`, `contenteditable`, `role="textbox"`) и отправить текст туда. Чтобы не попасть не в тот элемент, задай точные **селекторы** в панели и проверь их кнопкой **Проверить** на уже полностью загруженном чате.
