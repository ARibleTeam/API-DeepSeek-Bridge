# Локальный мост DeepSeek (расширение)

Расширение связывает браузерный DeepSeek и локальный Python-клиент по WebSocket.

## Что делает расширение

- Подключается к локальному WebSocket Python-клиента.
- Принимает задачи (`deepseek_task`) от Python.
- Открывает/использует вкладку DeepSeek, отправляет запрос и получает ответ.
- Возвращает результат в Python (`deepseek_task_result`).
- Поддерживает параллельное выполнение задач + очередь.

## Установка

1. Открой `chrome://extensions` (или `edge://extensions`).
2. Включи **Режим разработчика**.
3. Нажми **Загрузить распакованное расширение**.
4. Выбери папку `extension`.
5. Закрепи иконку расширения в панели браузера.

## Настройка в popup

По умолчанию видно только:

- `Адрес` (хост Python-сервера, обычно `127.0.0.1`)
- `Порт` (обычно `8765`)
- кнопка `Соединить`

### Поведение

- `Соединить`:
  - применяет адрес/порт;
  - включает мост и запускает подключение к Python;
  - после запуска адрес/порт становятся недоступны для изменения.
- `Отключить`:
  - отключает мост;
  - останавливает переподключение;
  - снова разрешает редактировать адрес/порт.

## Настройки селекторов DeepSeek

Секция селекторов скрыта по умолчанию.

Нажми **Показать настройки селекторов**, чтобы открыть:

- `Поле ввода`
- `Кнопка отправки`
- `Блок ответа`
- `Цитаты/источники`

### Кнопки

- `Проверить` - проверяет селекторы на открытой вкладке `https://chat.deepseek.com/*`.
- `Сохранить` - сохраняет селекторы в `chrome.storage.local`.
- `Сбросить` - возвращает селекторы по умолчанию.

### Подсветка при проверке

При `Проверить` расширение:

- выделяет найденные элементы яркой красной рамкой;
- показывает подписи на странице:
  - `поле ввода`
  - `кнопка отправки`
  - `блок ответа`
  - `цитаты`

### Что вставлять в поля

Вставляй **CSS-селектор**, а не весь HTML.

Как получить:

1. Открой DevTools -> `Elements`.
2. Выбери нужный элемент.
3. `Copy -> Copy selector`.
4. Вставь селектор в нужное поле.

Если у тебя только `outerHTML`, выдели в нем стабильные class/атрибуты и собери CSS-селектор.

## Параллельность и очередь

- Лимит одновременных задач: `3` (`MAX_PARALLEL_TASKS`).
- Если все слоты заняты, новые задачи ставятся в FIFO-очередь.
- При освобождении слота следующая задача запускается автоматически.

## Логи и отладка

Открой DevTools сервис-воркера расширения в `chrome://extensions` -> расширение -> **service worker**.

Ключевые логи:

- `[DeepSeek][ws] попытка подключения ...`
- `[DeepSeek][ws] подключено`
- `[DeepSeek][bridge] задача получена`
- `[DeepSeek][bridge] задача поставлена в очередь`
- `[DeepSeek][bridge] задача запущена`
- `[DeepSeek][bridge] результат отправлен в Python`
- `[DeepSeek][bridge] задача завершена`

## Формат обмена с Python

### Вход в расширение

```json
{
  "type": "deepseek_task",
  "requestId": "req-...",
  "payload": {
    "task": "Текст задачи",
    "timeoutMs": 180000
  },
  "timeoutMs": 180000
}
```

### Успешный ответ в Python

```json
{
  "type": "deepseek_task_result",
  "requestId": "req-...",
  "ok": true,
  "result": "Итоговый текст",
  "parsedJson": {},
  "sources": []
}
```

### Ответ с ошибкой

```json
{
  "type": "deepseek_task_result",
  "requestId": "req-...",
  "ok": false,
  "error": "deepseek_error",
  "message": "Описание ошибки"
}
```

## Где хранятся настройки

`chrome.storage.local`:

- `bridgeEnabled`
- `bridgeWsHost`
- `bridgeWsPort`
- `bridgeDeepseekSelectors`

## Ограничения

- Расширение работает с `https://chat.deepseek.com/*`.
- Для корректной работы пользователь должен быть авторизован в DeepSeek.
- Изменения в верстке DeepSeek могут требовать обновления селекторов.
